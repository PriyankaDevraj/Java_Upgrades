Enhancements in Java Language in Java SE 8

  • Lambda Expression(Not in SE7) – These are expression which encapsulates a functionality which can be executed instantly or passed on to other code.It enables us to treat functionality as method argument, or code as data. Example 1, Example 2

  • Method Reference(Not in SE7) - When a lambda expression invokes an existing method, we can use a method reference instead of lambda expression, as shown in example 2. Example 2

  • Default Methods(Not in SE7) - Default methods are methods defined in an interface with "default" keyword. These methods have default implementation in the interface.These methods are useful when we want to add a new functionality to an interface without impacting already written code.DefaultMethodInterface, DefaultMethodExample

  • Repeating Annotations(Not in SE7) - It provides the ability the same annotation type more than once to the same declaration or type use.AnnotationExample

  • Type Annotations(Not in SE7) - Annotations can be applied to any type use such as instance creation expression. In jave SE7 it could be applied only to declarations. AnnotationExample

  • Improved Type Inference - In Java SE 8 the type inference has been improved.This is illustrated in the example. TypeInferenceExample

  • Method parameter reflection -A new feature is provided in SE 8,in which one can get the names of formal parameters of any method or constructor. The java.lang.reflect package contains all the required classes like Method and Parameter to work with parameter reflection. Calculate ParameterReflection

  • For each loop - Java provides a new method forEach() to iterate the elements. It is defined in Iterable and Stream interface. ForEachExample

  • java.util.stream package - Java provides a new additional package in Java 8 called java.util.stream. This package consists of classes, interfaces and enum to allows functional-style operations on the elements.One can use stream to filter, collect, print, and convert from one data structure to other. StreamExample Person BulkDataOperationsExample

  • Optional Class - Java introduced a new class Optional in jdk8. It is a public final class and used to deal with NullPointerException in Java application. OptionalDemo

Enhancements in Java language in Java SE 9

  • Private Interface Methods – In Java 9, we can create private methods inside an interface. Interface allows us to declare private methods that help to share common code between non-abstract methods. PrivateInterfaceExample, Sayable

  • Try with resources – In java 7 try with resources was introduced in which one or more resources can be declared in the try statement as shown in FinalVariable.java.Try-with-resource is improved in Java 9 and now we can use reference of the resource that is not declared locally as shown in FinalVariable1.java. FinalVariable , FinalVariable1

  • Anonymous Inner Classes Improvement – In Java 9 diamond operator can be used with inner classes. This feature was not present in previous versions. AnonymousClassExample

  • @SafeVarargs Annotation improvement – @SafeVarargs were included in Java 7 and could be applied on final methods ,static methods and constructors.These annotations are used to ensure that safe operations are performed on varargs. In version 9 it is improved to be applied on private instance methods. SafeVar

  • Factory Methods – Java 9 Collection library includes static factory methods for List, Set and Map interface. These methods are useful to create small number of collection.They create unmodifiable i.e. immutable instances of collection. It is unmodifiable, so adding new element will throw java.lang.UnsupportedOperationException. FactoryMethodsExample

  • Process API Improvement –Java has improved its process API in Java 9 version that helps to manage and control operating system processes.In earlier versions, it was complex to manage and control operating system processes by using Java programming. Now, new classes and interfaces are added to perform this task. ProcessApiExample

  • Underscore Keyword - '_' should not be used as an identifier, since it is a reserved keyword from source level 1.8 on.UnderScoreExample

  • Version-String Scheme - In Java 9, a new version-string scheme is introduced that looks like - $MAJOR.$MINOR.$SECURITY.$PATCH .In Java 9, a Runtime.Version class is added to get Java version information. VersionInfoExample

  • Java Shell Tool (JShell) - It is an interactive Java Shell tool, it allows us to execute Java code from the shell and shows output immediately. JShell is a REPL (Read Evaluate Print Loop) tool and run from the command line.Jshell Snapshot

  • Java 9 Module System - Java Module System is a major change in Java 9 version. Java added this feature to collect Java packages and code into a single unit called module.o describe a module, a Java file module-info.java is required.

Enhancements in Java language in Java SE 11

  • New Collection.toArray(IntFunction) Default Method - A new default method toArray(IntFunction) has been added to the java.util.Collection interface. This method allows the collection's elements to be transferred to a newly created array of a desired runtime type.NewToArrayDemo

  • Local-Variable Syntax for Lambda Parameters -The reserved type name var can now be used when declaring the formal parameters of a lambda expression.If var is used for any formal parameter of a lambda expression, then it must be used for all formal parameters of that lambda expression.
